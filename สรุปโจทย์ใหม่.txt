📁 ไฟล์ที่สร้าง
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. โจทย์ใหม่_10_ข้อ_คำตอบ.txt
   - คำตอบที่ถูกต้องสำหรับทุกโจทย์
   - ตัวอย่างคำตอบที่ผิดพร้อมคำอธิบาย
   - จัดรูปแบบให้อ่านง่าย

2. insert_new_problems.sql
   - คำสั่ง SQL INSERT สำหรับเพิ่มโจทย์เข้า Supabase
   - เริ่มต้นด้วยคำสั่ง DELETE เพื่อล้างข้อมูลเก่า
   - เพิ่มโจทย์ ID 1-10

3. สรุปโจทย์ใหม่.txt (ไฟล์นี้)
   - สรุปภาพรวมของโจทย์ทั้งหมด
   - สถิติและการจัดหมวดหมู่
   - คำแนะนำในการใช้งาน

🚀 วิธีการใช้งาน
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ขั้นตอนที่ 1: เพิ่มโจทย์เข้า Supabase
────────────────────────────────────────────────────────────────────────────
1. เปิด Supabase Dashboard
2. ไปที่ SQL Editor
3. คัดลอกเนื้อหาจากไฟล์ insert_new_problems.sql
4. วางและรันคำสั่ง SQL (ระบบจะลบข้อมูลเก่าและเพิ่มโจทย์ใหม่)
5. ตรวจสอบว่าโจทย์ถูกเพิ่มเข้าตาราง Codecamp แล้ว

ขั้นตอนที่ 2: ทดสอบโจทย์
────────────────────────────────────────────────────────────────────────────
1. เปิดแอปพลิเคชัน Code Camp Academy
2. ไปที่หน้า Test Editor
3. เลือกโจทย์ใหม่ (ID 1-10)
4. ทดสอบด้วยคำตอบที่ถูกต้องจากไฟล์คำตอบ
5. ตรวจสอบว่าระบบตรวจคำตอบได้ถูกต้อง

ขั้นตอนที่ 3: ตรวจสอบคำตอบ
────────────────────────────────────────────────────────────────────────────
1. เปิดไฟล์ โจทย์ใหม่_10_ข้อ_คำตอบ.txt
2. ดูคำตอบที่ถูกต้องสำหรับแต่ละโจทย์
3. ศึกษาตัวอย่างคำตอบที่ผิดเพื่อเข้าใจข้อผิดพลาดทั่วไป
4. ใช้เป็นแนวทางในการสอนหรือตรวจสอบคำตอบของผู้เรียน

💡 คำแนะนำเพิ่มเติม
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

สำหรับผู้สอน:
  ✓ ใช้โจทย์ง่ายๆ (ระดับ 1-2) สำหรับผู้เริ่มต้น
  ✓ โจทย์ OOP (ระดับ 3) เหมาะสำหรับผู้ที่มีพื้นฐานแล้ว
  ✓ แนะนำให้ทำโจทย์ตามลำดับความยาก
  ✓ ใช้ตัวอย่างคำตอบที่ผิดเป็นกรณีศึกษา

สำหรับผู้เรียน:
  ✓ อ่านโจทย์ให้เข้าใจก่อนเริ่มเขียนโค้ด
  ✓ ทดสอบโค้ดด้วยตัวเองก่อนส่งคำตอบ
  ✓ ถ้าผิด ให้ดูตัวอย่างคำตอบที่ผิดเพื่อหาข้อผิดพลาด
  ✓ เรียนรู้จากความผิดพลาดและลองใหม่

สำหรับผู้พัฒนา:
  ✓ ตรวจสอบว่า expected_output ตรงกับผลลัพธ์จริง
  ✓ ทดสอบกับหลายๆ test case
  ✓ เพิ่ม validation_mode อื่นๆ ถ้าต้องการ (เช่น syntax_check)
  ✓ อัพเดทจำนวน likes ตามความนิยม

📌 หมายเหตุสำคัญ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️ การล้างข้อมูล (Reset)
   - ไฟล์ SQL มีคำสั่ง DELETE FROM "Codecamp"
   - ข้อมูลโจทย์เก่าทั้งหมดจะหายไป
   - จะเหลือเฉพาะโจทย์ใหม่ 10 ข้อนี้เท่านั้น

⚠️ ฟิลด์ที่สำคัญใน Supabase
   - id: รหัสโจทย์ (1-10)
   - title: ชื่อโจทย์
   - description: คำอธิบายโจทย์
   - language: ภาษาโปรแกรม
   - difficulty: ระดับความยาก (1-5)
   - expected_output: ผลลัพธ์ที่ต้องการ
   - initial_code: โค้ดเริ่มต้น
   - validation_mode: โหมดการตรวจสอบ
   - likes: จำนวนไลค์

⚠️ การทดสอบ
   - ทดสอบทุกโจทย์ก่อนใช้งานจริง
   - ตรวจสอบว่า expected_output ถูกต้อง
   - ทดสอบทั้งคำตอบที่ถูกและผิด
